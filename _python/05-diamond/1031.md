# BaekJoon 1031 - 스타와 선수 (최대 유량 + 사전 순 최적화)

## 문제 분석

### 문제 개요
- **지민이 팀**: N명, 각 선수가 해야 할 경기 수 주어짐
- **한수팀**: M명, 각 선수가 해야 할 경기 수 주어짐
- **목표**: N×M 크기의 대진표 구성 (1=경기, 0=미경기)
  - 모든 팀원이 정해진 경기 수만큼 경기해야 함
  - 같은 대결은 최대 1회
  - **사전 순으로 가장 앞선 대진표** 선택

### 사전 순 정의
두 대진표에서:
1. 첫 번째로 다른 행 i를 찾음
2. 그 행에서 첫 번째로 다른 열 j를 찾음
3. **(i,j)가 0인 대진표가 사전 순으로 앞서감**

즉, **가능한 한 많은 0을 배치하되, 왼쪽-위부터 우선**

## 해결 전략

### 핵심 아이디어: 이분 그래프 + 최대 유량 + 필수 간선 판정

```
┌─────────────────────────────────────────────┐
│ 1단계: 최대 유량 계산                        │
│ (모든 경기를 배치할 수 있는지 확인)          │
└─────────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────────┐
│ 2단계: 사전 순 정렬                          │
│ 각 간선(i,j)마다:                           │
│ - 간선 제거 후 대체 경로 검색                │
│ - 있으면 → 이 간선 불필수 (0)              │
│ - 없으면 → 이 간선 필수 (1)                │
└─────────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────────┐
│ 3단계: 결과 출력                             │
└─────────────────────────────────────────────┘
```

## 알고리즘 상세 설명

### 1단계: 최대 유량 계산 (Edmonds-Karp)

**네트워크 구성:**
```
Source(0)
    │
    ├─ 지민이 팀(1~N) ───[용량1]──→ 한수 팀(N+1~N+M)
    │                                        │
    │                                    Sink(N+M+1)
```

**간선:**
- Source → 지민 선수 i: 용량 = g_games[i]
- 지민 i → 한수 j: 용량 = 1 (최대 1경기)
- 한수 j → Sink: 용량 = h_games[j]

**Edmonds-Karp 알고리즘:**
```python
def max_flow(source, sink):
    while BFS로 증강경로 찾기:
        최소용량 = 경로 상의 최소 잔여 용량
        경기 배치 += 최소용량
        경로 상 유량 업데이트:
            - 포워드 간선: 유량 증가
            - 역방향 간선: 유량 감소 (흐름 조정 가능)
    return 총 배치 경기 수
```

**시간복잡도:** O(VE²) = O((N+M) × (N×M)²)

### 2단계: 사전 순 최적화 (필수 간선 판정)

**아이디어:**
- 각 간선 (i,j)를 왼쪽-위부터 순회
- 간선을 제거했을 때, **사전 순으로 그 뒤의 간선들만 사용해서** 대체 경로가 있는지 확인
- **있으면** → 이 간선은 불필수 (0으로 설정 가능)
- **없으면** → 이 간선은 필수 (1로 설정해야 함)

**예시:**
```
대진표 (i=0, j=0) 시점:
D[0][0] = 1 (현재 배치)

1) 간선 (0,0) 제거 → flow[0][0] = 0
2) (0,0)보다 뒤의 간선들만 사용해서 흐름 재구성 가능?
   - Case A: 가능 → (0,0) = 0 유지
   - Case B: 불가능 → (0,0) = 1 필수
```

### 3단계: 대체 경로 탐색 (BFS)

**사전 순 조건을 만족하는 경로 탐색:**

```python
def can_reroute_bfs(u, target, start_i, start_j, n_g, n_h):
    """u → target 경로를 (start_i, start_j)보다 뒤의 간선만 사용해서 찾기"""
    
    visited = {u}
    queue = [u]
    
    while queue not empty:
        curr = queue.pop()
        
        if curr == target:
            return 경로 복원
        
        if curr는 지민팀:
            for 한수팀의 선수 v:
                if (curr-1, v-n_g-1)가 (start_i, start_j)보다 뒤 and 간선 가능:
                    탐색 계속
        else:
            for 지민팀의 선수 v:
                if (v-1, curr-n_g-1)이 (start_i, start_j)보다 뒤 and 간선 가능:
                    탐색 계속
```

## 코드 상세 분석

### 클래스: EdmondsKarp

#### 초기화
```python
def __init__(self, n: int):
    self.capacity = [[0] * n for _ in range(n)]  # 용량 행렬
    self.flow = [[0] * n for _ in range(n)]      # 현재 유량 행렬
```

#### BFS (증강경로 탐색)
```python
def bfs(self, source: int, sink: int) -> list:
    parent = [-1] * self.n
    parent[source] = source
    queue = deque([source])
    
    while queue:
        u = queue.popleft()
        for v in range(self.n):
            # 미방문 + 잔여 용량 > 0
            if parent[v] == -1 and self.capacity[u][v] - self.flow[u][v] > 0:
                parent[v] = u
                if v == sink:
                    # 경로 복원
                    return 경로_복원(parent, sink, source)
                queue.append(v)
    
    return []  # 경로 없음
```

**특징:**
- BFS이므로 최단 증강경로 탐색 → Edmonds-Karp 보장
- `capacity[u][v] - flow[u][v] > 0`: 잔여 용량 확인

#### 최대 유량
```python
def max_flow(self, source: int, sink: int) -> int:
    total = 0
    
    while True:
        path = self.bfs(source, sink)
        if not path:
            break
        
        # 경로의 병목(최소 용량) 찾기
        min_flow = min(capacity[u][v] - flow[u][v] for (u,v) in path)
        
        # 유량 업데이트
        total += min_flow
        for i in range(len(path) - 1):
            u, v = path[i], path[i+1]
            flow[u][v] += min_flow      # 포워드 에지
            flow[v][u] -= min_flow      # 백워드 에지 (흐름 조정)
    
    return total
```

#### 대체 경로 탐색 (최적화된 BFS)
```python
def can_reroute_bfs(self, u, target, start_i, start_j, n_g, n_h):
    parent = {u: None}
    queue = deque([u])
    
    while queue:
        curr = queue.popleft()
        
        if curr == target:
            return 경로복원()
        
        if curr <= n_g:  # 지민 팀
            for v in range(n_g+1, n_g+n_h+1):  # 한수 팀만
                if v not in parent:
                    i, j = curr-1, v-n_g-1
                    # 사전순 조건 & 용량 확인
                    if (i > start_i or (i == start_i and j > start_j)) \
                       and capacity[curr][v] - flow[curr][v] > 0:
                        parent[v] = curr
                        queue.append(v)
        else:  # 한수 팀
            for v in range(1, n_g+1):  # 지민 팀만
                if v not in parent:
                    i, j = v-1, curr-n_g-1
                    if (i > start_i or (i == start_i and j > start_j)) \
                       and capacity[curr][v] - flow[curr][v] > 0:
                        parent[v] = curr
                        queue.append(v)
    
    return None  # 경로 없음
```

**최적화:**
- 지민 팀 노드 → 한수 팀만 순회 (O(n) 대신 O(m))
- 한수 팀 노드 → 지민 팀만 순회 (O(m) 대신 O(n))
- 전체 순회 최대 O(n+m) → **반복 횟수 ~50% 감소**

### 메인 함수 흐름

#### 1단계: 입력 & 유효성 검사
```python
n, m = map(int, input().split())
g_games = list(map(int, input().split()))  # 지민 팀
h_games = list(map(int, input().split()))  # 한수 팀

# 총 경기 수 일치 확인
if sum(g_games) != sum(h_games):
    print(-1)
    return
```

#### 2단계: 네트워크 생성 & 최대 유량 계산
```python
ek = EdmondsKarp(n + m + 2)  # 노드: 0(src), 1~n(지민), n+1~n+m(한수), n+m+1(sink)

# 간선 추가
for i in range(n):
    ek.add_edge(source, i+1, g_games[i])  # Source → 지민팀

for i in range(n):
    for j in range(m):
        ek.add_edge(i+1, n+1+j, 1)  # 지민팀 → 한수팀

for j in range(m):
    ek.add_edge(n+1+j, sink, h_games[j])  # 한수팀 → Sink

# 최대 유량 계산
max_flow_value = ek.max_flow(source, sink)
if max_flow_value != sum(g_games):
    print(-1)
    return
```

#### 3단계: 사전 순 정렬 (필수 간선 판정)
```python
result = [[0] * m for _ in range(n)]

for i in range(n):
    for j in range(m):
        u = i + 1
        v = n + 1 + j
        
        if ek.flow[u][v] == 1:  # 현재 배치된 간선
            # 1. 간선 제거
            ek.flow[u][v] = 0
            ek.flow[v][u] = 0
            
            # 2. 대체 경로 탐색 (i,j보다 뒤의 간선만 사용)
            path = ek.can_reroute_bfs(u, v, i, j, n, m)
            
            if path:  # 대체 경로 있음
                # 경로를 따라 유량 업데이트
                for k in range(len(path) - 1):
                    a, b = path[k], path[k+1]
                    ek.flow[a][b] += 1
                    ek.flow[b][a] -= 1
                result[i][j] = 0  # 불필수
            else:  # 대체 경로 없음
                # 간선 복구
                ek.flow[u][v] = 1
                ek.flow[v][u] = -1
                result[i][j] = 1  # 필수
```

## 예시 실행

### 예제 1: 3x3 대진표
```
입력:
3 3
1 2 3
3 1 2

분석:
- 지민: [1, 2, 3] (총 6경기)
- 한수: [3, 1, 2] (총 6경기)
- 경기 수 일치 ✓

출력:
100  (지민선수0은 한수선수0과만 경기)
101  (지민선수1은 한수선수0,2와 경기)
111  (지민선수2는 한수선수0,1,2와 경기)
```

**과정:**
```
(0,0): 전체 배치 후, (0,0)을 0으로 유지 가능? 
       YES → 1은 불필수, 결과는 0이 아니라... 
       아 잠깐, flow[0][0] = 1이니까 (0,0)에 경기가 있다는 뜻.
       제거 후 대체경로 있는지 확인 → 있으면 0 유지

실제로는 초기 max_flow에서 결정된 배치가:
D[0][0]=1, D[1][0]=1, D[1][2]=1, D[2][0]=1, D[2][1]=1, D[2][2]=1

(0,0) 체크: 제거 후 대체경로 있나? 
- 한수팀 선수0의 용량은 3 필요인데, 다른 경로로 2개만 가능
- 대체경로 없음 → 필수 (1 유지)

(0,1) 체크: 제거해도 괜찮나?
- 한수팀 선수1의 흐름을 다시 구성 가능
- 대체경로 있음 → 불필수 (0)
```

## 시간복잡도 분석

| 단계 | 복잡도 | 설명 |
|------|--------|------|
| 최대 유량 (Edmonds-Karp) | O(VE²) | V=N+M+2, E=N×M |
| 필수 간선 판정 | O(N×M × BFS) | N×M개 간선, 각각 BFS |
| BFS 최적화 | O(N+M) | 이분 그래프 특성 활용 |
| **전체** | **O(N×M×(N+M)²)** | 대략 50² × 100² ~ 25,000,000 |

**주어진 제약:**
- N, M ≤ 50
- 최악: 50 × 50 × 100² = 25,000,000 연산
- 실제: BFS 조기 종료로 더 빠름

## 핵심 최적화 기법

### 1. 이분 그래프 특성 활용
```python
if curr <= n_g:
    for v in range(n_g+1, n_g+n_h+1):  # 한수팀만 탐색
else:
    for v in range(1, n_g+1):  # 지민팀만 탐색
```
→ 노드 순회 범위 ~50% 감소

### 2. 사전 순 조건 조기 필터링
```python
if i > start_i or (i == start_i and j > start_j):
    # 이 간선 탐색 허용
```
→ 불필요한 간선 탐색 제외

### 3. BFS로 재귀 깊이 제한
- DFS: 스택 오버플로우 위험
- BFS: 최단 경로 보장, 안정적
